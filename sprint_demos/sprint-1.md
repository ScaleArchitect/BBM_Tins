## 4. Result — RLS isolation (live, as `tin_app`)

```
 connected_as | is_superuser
--------------+--------------
 tin_app      | f

 context  | visible |     rows
----------+---------+---------------
 ACME ctx |       1 | admin@acme.ae       (SET app.current_company_id = ACME)

  context   | visible |      rows
------------+---------+-----------------
 GLOBEX ctx |       1 | admin@globex.ae    (SET app.current_company_id = GLOBEX)

 context | visible | rows
---------+---------+------
 no ctx  |       0 | -                    (RESET -> deny by default)
```

A tenant sees **only** its own `company_admins`; with no tenant context set, it
sees **nothing** (deny by default). The owner/superuser path bypasses RLS by
design for platform operations.

## 6. Issues found & fixed during the demo

| # | Issue | Fix |
|---|-------|-----|
| 1 | After `RESET`, the RLS predicate errored: `invalid input syntax for type uuid: ""` — an empty GUC was cast to uuid | Wrapped the GUC in `NULLIF(current_setting(...), '')::uuid` so empty/unset → NULL → 0 rows (deny by default) |
| 2 | Autogenerated migration did not drop enum types on downgrade (breaks re-upgrade) | Added explicit `DROP TYPE` statements to `downgrade()` |

## 7. Reproduce

```bash
docker compose -f infra/compose/docker-compose.yml up -d --build

# RLS proof as the app role
docker compose -f infra/compose/docker-compose.yml exec -e PGPASSWORD=tin_app db \
  psql -U tin_app -h 127.0.0.1 -d tin -c \
  "SET app.current_company_id='<company-uuid>'; SELECT count(*) FROM company_admins;"

# tests
cd backend && pytest -q
```
